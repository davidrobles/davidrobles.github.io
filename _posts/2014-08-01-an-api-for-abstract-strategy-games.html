In this post I'll describe a simple API for abstract strategy games and algorithms to play these games. Almost all abstract strategy games conform to the strictest definition of: a gameboard, card, or tile game in which there is no hidden information, no non-deterministic elements, and (usually) two players or teams taking a finite number of alternating turns. However, the API I'll describe works for games and puzzles that consist of any number of players taking a finite number of alternating turns.

Here are a few games that can integrate with this API:

  • Eight queens puzzle (1 Player)
  • Tic-Tac-Toe (2 Players)
  • Chess (2 Players)
  • Othello (2 Players)
  • Chinese Checkers (2–4 or 6 Players)

The game of Tic-Tac-Toe will be used as an example to describe the functionality of the API. Tic-Tac-Toe was chosen because is one of the simplest, yet most popular abstract strategy board games, and most people are familiar with the rules. The API consists of two interfaces that will be described next:

  • Game
  • Player

Game

What is a game? From the game theoretical point of view a game is a description of strategic interaction that includes the constraints on the actions that the players can take and the players’ interests, but does not specify the actions that the players do take (Rasmusen, 2006). Based on this definition, we can define an interface for abstract strategy games as follows:

  • .copy() Returns a copy of the game.
  • .currentPlayer() Returns the player whose turn it is.
  • .isGameOver() Returns true if the game is over, false otherwise.
  • .move(move) Makes a move for the player whose turn it is.
  • .moves() Returns an Array of legal moves for the player in turn.
  • .newGame() Returns a new game.
  • .outcomes() Returns the outcome for each player.
  • .restart() Restarts the game.

Now here is a more detailed description of each of these methods with examples of their behavior.

.copy()

Returns a copy of the game.

Example {% highlight javascript %} var tic = new TicTacToe(); tic.move("5"); tic.move("9"); render(tic); {% endhighlight %} {% highlight javascript %} var ticCopy = tic.copy(); render(ticCopy); {% endhighlight %}
.currentPlayer()

Returns the player whose turn it is.

Example

{% highlight javascript %} var tic = new TicTacToe(); render(tic); {% endhighlight %} {% highlight javascript %} tic.currentPlayer(); // "1" tic.move("5"); render(tic); {% endhighlight %} {% highlight javascript %} tic.currentPlayer(); // "2" {% endhighlight %}
.isGameOver()

Returns true if the game is over, false otherwise.

Example

{% highlight javascript %} var tic = new TicTacToe(); render(tic); {% endhighlight %} {% highlight javascript %} tic.isGameOver(); // false tic.move("5"); tic.move("4"); tic.move("1"); tic.move("9"); tic.move("3"); tic.move("2"); tic.move("7"); render(tic); {% endhighlight %} {% highlight javascript %} tic.isGameOver(); // true {% endhighlight %}
.move(move)

Makes a move for the player whose turn it is.

Example: {% highlight javascript %} var tic = new TicTacToe(); render(tic); {% endhighlight %} {% highlight javascript %} tic.move("5"); render(tic); {% endhighlight %}
.moves()

Returns an Array of legal moves for the player in turn.

Example: {% highlight javascript %} var tic = new TicTacToe(); render(tic); {% endhighlight %} {% highlight javascript %} tic.moves(); // ["1", "2", "3", "4", "5", "6", "7", "8", "9"] tic.move("5"); tic.move("4"); tic.move("1"); render(tic); {% endhighlight %} {% highlight javascript %} tic.moves(); // ["2", "3", "6", "7", "8", "9"] tic.move("9"); tic.move("3"); tic.move("2"); tic.move("7"); render(tic); {% endhighlight %} {% highlight javascript %} tic.moves(); // [] {% endhighlight %}

.newGame()

Returns a new game.

Example: {% highlight javascript %} var tic = new TicTacToe(); tic.move("5"); tic.move("9"); render(tic); {% endhighlight %} {% highlight javascript %} var newGame = tic.newGame(); render(newGame); {% endhighlight %} {% highlight javascript %} render(tic); {% endhighlight %}
.outcomes()

Returns a dictionary with the outcomes for each of the players.

{% highlight javascript %} var tic = new TicTacToe(); render(tic); {% endhighlight %} {% highlight javascript %} tic.move("5"); tic.move("4"); tic.move("1"); tic.move("9"); tic.move("3"); tic.move("2"); tic.move("7"); render(tic); {% endhighlight %} {% highlight javascript %} tic.isGameOver(); // true tic.outcomes(); // { "1": "WIN", "2": "LOSS" } {% endhighlight %}
.restart()

Restarts the game.

Example

{% highlight javascript %} var tic = new TicTacToe(); tic.move("5"); tic.move("9"); render(tic); {% endhighlight %} {% highlight javascript %} tic.restart(); render(tic); {% endhighlight %}

Player

Players are the individuals who make decisions (making moves). We can define an interface for a player as simple as this: {% highlight javascript %} function player(game) { return "Take Move 1"; } {% endhighlight %} A player is asked for the move to be taken and is provided with the current state of the game. The player can analyze the current state of the game (e.g. search algorithm) and return the move to be taken.

Random Player

We can easily create a uniformly random player that can be used in any game that follows the Game interface as follows: {% highlight javascript %} function randomPlayer(game) { return Math.floor(Math.random() * game.moves().length); } {% endhighlight %}

Play a Match

Now that we have described the Game and Player interfaces we can use them to play a full match of Tic-Tac-Toe: {% highlight javascript %} var tic = new TicTacToe(); render(tic); {% endhighlight %} {% highlight javascript %} var tic = new TicTacToe(); while (!tic.isGameOver()) { tic.move(randomPlayer(tic)); } {% endhighlight %}

Summary

The API presented in this post is very simple, yet very concise to implement games and create game-playing algorithms that can play in games that follow the Game interface. Evidently, to find the optimal (or near optimal) moves the algorithms will use game-specific heuristics, however, if they the actual algorithms don't need to change much as long as the heuristics are independent of the behaviour of the algorithms.

Defining a Game interface like the one before is the base of a game-playing framework and it will be used in following blog posts describing game-playing algorithms such as Minimax, Alpha-Beta and Monte-Carlo Tree Search.
